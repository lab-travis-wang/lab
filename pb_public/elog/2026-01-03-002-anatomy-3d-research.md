# 实验002 - 人体骨骼肌肉3D可视化 调研记录

## 任务目标

为健康记录产品 UI 添加人体骨骼肌肉3D可视化功能，方便用户记录具体身体问题。

## 调研过程

### 1. 搜索可用资源

调研了以下几种方案：

| 方案 | 来源 | 许可 | 质量 |
|------|------|------|------|
| Sketchfab 模型 | l.kuzyakin | 需购买 | 高 |
| **AnatomyTOOL** | 荷兰莱顿大学等7所大学 | **CC BY-SA 4.0** | 学术级 |
| Visible Human Project | 美国国立医学图书馆 | 公共领域 | 高 |
| BioDigital Human | 商业公司 | 付费 | 高 |

### 2. AnatomyTOOL 深度评估

**项目背景：**
- 开发机构：莱顿大学、乌得勒支大学、马斯特里赫特大学、鲁汶大学等
- 许可证：Creative Commons Attribution ShareAlike 4.0 (可商用)
- 最新更新：2025年7月

**模型结构分析（上肢模型）：**
```
├── 🦴 骨骼 (26个节点)
├── 💪 肌肉 (46个节点)
├── 🧠 神经 (51个节点 - 臂丛神经、尺神经等)
├── 🩸 血管 (61个节点 - 动脉+静脉)
├── 🔗 韧带/筋膜
└── 🛡️ 软骨
```

**已下载测试的模型：**
- `overview-skeleton.glb` - 3.3MB (完整骨骼，147节点，144网格)
- `upper-limb.glb` - 6.6MB (上肢完整解剖，575节点)
- `lower-limb.glb` - 5.9MB (下肢)

### 3. 技术可行性分析

**✅ 可实现的功能：**
- 骨骼3D展示
- 上肢/下肢肌肉、神经、血管分层
- 射线检测交互 (Raycasting)
- 问题记录表单

**⚠️ 部分限制：**
- 模型仅包含右侧身体（左侧已镜像，只需程序化复制）
- 无皮肤层模型（计划2026年发布）
- 无内脏层模型（计划2026年发布）
- DRACO压缩需额外配置

**❌ 当前不可用：**
- 完整皮肤层
- 完整内脏系统

### 4. 任务复杂度评估

| 阶段 | 功能 | 预估工时 |
|------|------|----------|
| 镜像生成完整身体 | 将右侧模型复制并镜像为左侧 | 2-3小时 |
| 分层切换系统 | 骨骼/肌肉/神经/血管独立显示 | 4-6小时 |
| 交互高亮 | 点击部位高亮、显示名称 | 3-4小时 |
| 皮肤/内脏层 | 需等待2026年 AnatomyTOOL 更新 | 待定 |

## 决策结论

**暂停实验002的3D模型开发**

原因：
1. **复杂度超出预期**：需要处理大量解剖结构节点
2. **资源不完整**：皮肤、内脏层尚未发布
3. **优先级调整**：当前应聚焦核心功能

## 后续建议

如需继续开发，建议：

1. **等待 AnatomyTOOL 更新** (2026年)
   - 届时将包含完整皮肤、内脏系统
   - 可获得完整的人体解剖模型

2. **简化方案**
   - 使用2D解剖图替代
   - 或使用 Sketchfab 嵌入现成的交互式模型
